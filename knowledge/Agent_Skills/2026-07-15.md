## Cases

### CASE: triviality-gate-over-significance-gate
status: auto
problem: The min-turns capture gate counts only genuine user prompts, so an agentic session with 3–4 prompts and hours of engineering work is skipped as trivial.
decision: v3.1 replaces the significance proxy with a triviality gate — skip the sweep only when every available deterministic signal proves the session was trivial; otherwise run.
why: Error costs are asymmetric — a false skip silently and permanently loses a session's cases (the failure v3 exists to eliminate), while a false run costs one cheap model call that outputs NOTHING; user-turn (A), transcript-size (B), activity-signal (C), and multi-factor (D) designs were weighed and any deterministic significance proxy was judged doomed to break with workflow drift.
tags: [capture-gate, architecture, heuristics, hooks]

### CASE: repair-missing-front-matter-close-over-reject
status: auto
problem: The first live v3.1 sweep in issue-runtime failed because the model returned a complete, well-formed 9KB day file missing exactly one line — the closing `---` of the front matter — and the validator rejected the whole output.
decision: v3.1.1 adds a validator repair path — a lone opening `---` followed by 2+ key:value lines gets the missing closing delimiter inserted (one attempt, structural-only, logged as `repaired=inserted-missing-front-matter-close`), then re-validates; SWEEP.md Rule 8 now demands the closing delimiter explicitly.
why: Rejecting a 99%-perfect output loses a session over one line that the prompt itself under-specified — Rule 8 hammered the opening delimiter and never mentioned the closing one.
tags: [validator, front-matter, sweep, recovery]
